# 社区行为准则

## 目标

本项目希望形成一个专业、克制、可验证的协作环境。讨论应围绕真实产品问题、Agent 行为、测试证据和可维护性展开，不因观点、背景或经验差异攻击参与者。

## 适用范围

本准则适用于仓库的 Issue、Pull Request、Discussion、代码评审，以及代表本项目参与的其他公开或私下交流。

## 期望行为

- 尊重不同经验、行业、地区和技术背景。
- 对方案提出具体、可验证的批评，不对人作判断。
- 明确区分事实、假设、个人经验和价值取向。
- 主动删除公司机密、用户隐私、账号密钥和未授权数据。
- 接受维护者基于项目边界、证据质量和维护成本作出的取舍。
- 发现自己的判断有误时，及时更正并保留必要的上下文。

## 不可接受行为

- 侮辱、骚扰、威胁、歧视或人身攻击。
- 持续进行与具体产品工作无关的政治争论、历史争论或人物崇拜。
- 借 Issue、PR 或评论发布广告、引流信息和重复推广内容。
- 公开他人的个人信息、公司内部数据或私下沟通内容。
- 伪造测试、用户反馈、数据结果、Star、下载量或其他社区信号。
- 在维护者明确要求停止后继续干扰讨论或规避管理措施。

## 报告与处理

一般问题可以通过仓库 Issue 报告。涉及骚扰、隐私或其他不适合公开的信息时，请通过 [维护者 GitHub 主页](https://github.com/atdy) 提供的联系方式私下联系；不要在公开 Issue 中粘贴敏感内容。

维护者会在合理时间内确认报告、保存必要证据并评估影响。根据严重程度和历史记录，可采取内容更正、正式警告、临时限制参与或永久禁止参与等措施。处理过程会尽可能保护报告者和受影响人员的隐私。

## 归属

本准则参考 [Contributor Covenant 2.1](https://www.contributor-covenant.org/version/2/1/code_of_conduct/) 并结合本项目的产品决策与数据隐私场景进行了调整。
