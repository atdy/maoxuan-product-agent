# 前向测试报告

日期：2026-07-10

运行环境：Claude Code 2.1.201

安装方式：全新临时项目的 `.claude/skills/product-decision-agent/`

上下文：每条测试使用独立的 `claude -p` 调用，不提供预期答案或项目设计结论。

## 验收标准

- 产品问题应得到中文、直接、可执行的判断。
- 不暴露原文、人物、历史语境或后台方法名。
- 不把相关性直接当因果，能区分证据强弱。
- 信息不足时先给条件判断，再提出不超过 3 个关键问题。
- 非产品类原文解释问题不应被本 Skill 截获。

## 测试结果

| ID | 调用方式 | 场景 | 关键表现 | 结果 |
|---|---|---|---|---|
| FT-01 | 显式 `/product-decision-agent` | 社区 DAU 下滑，团队想跟竞品做全站红包 | 拒绝立即跟进；把竞品影响降为待验证假设；给出 48 小时诊断、分层和分支决策 | PASS |
| FT-02 | 隐式触发 | 增长停滞，投放、裂变、会员、内容四选一 | 先按增长链路判断卡点；指出留存不足时不要放大拉新；给 3 个会改变判断的确认项 | PASS |
| FT-03 | 应不触发 | 要求解释《矛盾论》并引用原文 | 正常完成原文解释，没有套用产品决策输出或拒绝回答 | PASS |
| FT-04 | 显式 `/product-decision-agent` | 新版上线后 D1 留存下降，同时渠道结构剧变且有少量客服反馈 | 用渠道占比复算结构稀释；把 8 条反馈视为方向性证据；给不回滚、局部处理、回滚的条件 | PASS |

## 观察

- FT-04 证明“主导机制与变化条件”会实际改变决策：Agent 没有因为时间上同时发生就建议全量回滚。
- FT-02 的隐式回答没有完全使用固定标题，但保留了判断、行动、停止项和必要确认。固定结构不应压过自然表达。
- 四条产品回答均未暴露方法来源；应不触发案例也没有受到产品 Skill 的边界污染。

## 结论

本轮没有发现需要阻止发布的行为问题。后续修改触发描述、证据分级或输出结构时，应重复 FT-02、FT-03、FT-04。
