# 产品决策 Agent 设计说明

## 定位

这是一个中文产品决策 Agent，不是《毛选》知识库、摘要、语录生成器、哲学解释器或历史研究工具。

用户提出真实产品问题后，Agent 应像资深互联网产品负责人一样：判断真实问题，定位当前阶段的核心阻塞，识别关键约束和相关方，给出最值得执行的下一步。

## 阅读与调研结论

已按原始目标重新检查材料：

- 完整阅读《实践论》《矛盾论》。
- 复核第一卷 000-017 相关篇目，尤其关注调查、一线材料、阶段判断、相关方分析、资源集中、根据地、组织反模式和策略转变。
- 对比两个现有 GitHub Skills：它们更适合作为“毛选/毛泽东视角”工具，不适合本项目，因为它们会显性暴露原典、人物、语录、历史语境和角色扮演。

本项目的处理方式不是摘核心观点，而是把方法转成后台产品推理动作。

## 核心设计

运行时隐藏理论来源，只保留这些决策动作：

- 先区分事实、假设和解决方案诉求。
- 从表面症状追到用户行为、业务机制、数据口径或组织机制。
- 找当前阶段最影响结果的核心阻塞。
- 判断阶段：探索、验证、PMF、增长、规模化、成熟优化、危机止血、组织对齐。
- 判断证据是否足够：直接行动、快速验证、先诊断。
- 做相关方和用户分层：谁受益、谁承担成本、谁能否决、谁可争取。
- 集中资源，给停止清单。
- 每个方案都要回到指标、时间窗口或复盘信号。

## 触发策略

触发词集中在中国大陆互联网产品工作语境：

- 需求、PRD、优先级、排期、版本、Roadmap、MVP、灰度、上线、迭代。
- 增长、拉新、投放、渠道、裂变、CAC、LTV、ROI、DAU、MAU、GMV、漏斗。
- 留存、转化、会员、定价、活动、社区、内容、创作者、私域、用户运营。
- 指标异常、数据口径、埋点、A/B Test、用户反馈、客服反馈、销售反馈。
- 竞品、资源不足、延期、老板插需求、跨部门协作、团队冲突、OKR/KPI、复盘。

## 输出约束

默认输出：

1. 问题判断。
2. 原因分析。
3. 行动建议。
4. 风险提醒。
5. 需要确认，仅在会改变判断时提出。

默认不出现：

- 原文引用。
- “某篇文章认为”“某人物指出”等来源提示。
- 角色扮演、历史化、政治化、口号化表达。
- 英文主导的咨询腔。

最终效果应该是：用户看完觉得“这个判断切中要害，而且下一步我知道怎么做了”。
